# Step 6: 실전 GraphRAG 시스템

## 개요
모든 것을 종합하여 실전에서 사용 가능한 GraphRAG 시스템을 구축합니다.

## GraphRAG 시스템 아키텍처

```
문서 입력
    ↓
┌─────────────────┐
│ 인덱싱 파이프라인 │
│ - 청킹          │
│ - 엔티티 추출   │
│ - 관계 추출     │
│ - 그래프 구축   │
│ - 커뮤니티 감지 │
│ - 요약 생성     │
└─────────────────┘
    ↓
┌─────────────────┐
│  지식 베이스    │
│ - 벡터 DB       │
│ - 그래프 DB     │
│ - 커뮤니티 요약 │
└─────────────────┘
    ↓
사용자 질문
    ↓
┌─────────────────┐
│ 검색 엔진       │
│ - 벡터 검색     │
│ - 그래프 탐색   │
│ - 하이브리드    │
└─────────────────┘
    ↓
LLM 답변 생성
```

## 검색 전략

### 1. Vector Search
- 빠름
- 의미적 유사성
- 단일 홉

### 2. Graph Search
- 구조적 관계
- 다중 홉 추론
- 전역 이해

### 3. Hybrid Search (추천)
- 둘의 장점 결합
- 최고의 성능

## RAG vs GraphRAG

| 측면 | 기본 RAG | GraphRAG |
|------|----------|----------|
| 검색 | 벡터 유사도만 | 벡터 + 그래프 |
| 추론 | 단일 문서 | 다중 홉 |
| 전역 질문 | 어려움 | 커뮤니티 요약 활용 |
| 복잡도 | 낮음 | 높음 |
| 성능 | 좋음 | 우수 |

## 실전 활용

### 사용 사례
1. 복잡한 도메인 지식 (법률, 의료)
2. 다중 문서 추론
3. 전역적 질문 ("전체 프로젝트 구조는?")
4. 관계 기반 검색

### 성능 최적화
- 그래프 크기 제한
- 커뮤니티 수 조절
- 캐싱 활용
- 병렬 처리
