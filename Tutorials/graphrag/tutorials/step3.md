# Step 3: 그래프 구조 구축

## 개요
GraphRAG의 핵심은 문서를 그래프로 표현하는 것입니다.
엔티티(노드)와 관계(엣지)를 추출하여 지식 그래프를 구축합니다.

## 실행
```bash
python tutorials/step3.py
```

## 그래프 기반 RAG의 장점

**전통적 RAG:**
```
문서 → 청크 → 벡터 → 유사도 검색
```
- 단순 텍스트 유사도만 고려
- 문서 간 관계 무시

**GraphRAG:**
```
문서 → 엔티티 + 관계 → 그래프 → 그래프 쿼리
```
- 엔티티 간 관계 활용
- 다중 홉(multi-hop) 추론
- 전역적 문맥 이해

## 핵심 개념

### 엔티티 (Entity)
- 사람, 조직, 개념, 제품 등
- 그래프의 노드

### 관계 (Relationship)
- 엔티티 간 연결
- 그래프의 엣지
- 예: WORKS_FOR, DEVELOPED, PART_OF

### 지식 그래프 (Knowledge Graph)
- 엔티티와 관계의 네트워크
- 도메인 지식을 구조화
